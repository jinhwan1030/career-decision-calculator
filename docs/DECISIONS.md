# Decisions

## 2026-06-19: Vite + React + TypeScript 선택

MVP가 작은 로컬 우선 계산기이므로 Next.js를 쓰지 않고 Vite 기반 SPA로 시작한다.

## 2026-06-19: 서버와 로그인 제외

개인 연봉, 퇴사일, 연차 정보가 포함되므로 MVP에서는 데이터를 외부 전송하지 않는다. localStorage만 사용한다.

## 2026-06-19: 계산 로직과 UI 분리

계산 정확성과 문서화를 위해 보상, 출퇴근, 퇴사일 계산을 `src/lib/calculations`에 둔다.

## 2026-06-19: 퇴직금과 연차수당은 참고 계산으로 제한

법정 계산기나 노무 자문으로 오해되지 않도록 단순 추정치와 주의 문구를 함께 표시한다.

## 2026-06-19: JSON export/import는 텍스트 기반으로 시작

MVP에서는 서버나 파일 저장 API를 쓰지 않고, 사용자가 직접 JSON을 복사해 옮길 수 있는 구조로 시작한다. 이후 UX가 필요하면 파일 선택과 다운로드 버튼을 추가한다.

## 2026-06-19: 서비스 워커는 프로덕션에서만 등록

개발 중 캐시가 UI 확인을 방해하지 않도록 `import.meta.env.PROD`일 때만 서비스 워커를 등록한다.

## 2026-06-19: 계산 가정값은 이직 비교 화면에서 직접 편집

시간가치, 재택근무 가치, 리스크 버퍼는 사용자마다 크게 달라진다. MVP에서도 기본값만 고정하지 않고 결과 근처에서 조정할 수 있게 한다.

## 2026-06-19: 퇴사월 월급 정산 기준은 두 가지로 제한

MVP에서는 30일 기준 일급과 퇴사월 실제 달력일 기준만 제공한다. 회사별 급여기간, 무급휴가, 지각/조퇴, 공제 기준은 안내 문구와 계산 한계로 남긴다.

## 2026-06-19: 인쇄용 스타일은 CSS로 우선 처리

PDF 생성이나 이미지 생성은 MVP 제외 범위이므로 별도 export 기능 대신 브라우저 인쇄가 읽기 좋게 나오도록 CSS print 스타일을 먼저 제공한다.

## 2026-06-19: 결과 공유는 브라우저 기본 API로 제한

외부 공유 SDK를 쓰지 않는다. 지원 브라우저에서는 Web Share API를 사용하고, 그렇지 않으면 클립보드 복사로 대체한다.

## 2026-06-19: import JSON은 구조 검증 후 저장

localStorage로 덮어쓰기 전에 `scenarios` 배열과 시나리오 기본 필드를 확인한다. 세부 계산 필드 검증은 이후 마이그레이션 단계에서 강화한다.

## 2026-06-19: 공유 전 문구를 사용자가 확인할 수 있게 함

공유 결과가 단정적 조언처럼 보이지 않도록 실제 공유될 문구를 결과 화면에서 미리 확인할 수 있게 한다.

## 2026-06-19: 시나리오 제목은 저장 전 직접 편집 가능

회사명 조합으로 자동 제목을 만들되, 사용자가 대시보드에서 찾기 쉬운 이름으로 바꿀 수 있게 한다.

## 2026-06-19: 결제와 과금 요소 제외

MVP는 완전한 로컬 우선 계산기이며 결제 SDK, 구독, 광고, 분석 SDK를 포함하지 않는다.

## 2026-06-19: JSON import 중복 ID는 새 ID로 보정

가져온 JSON 안에 같은 시나리오 ID가 여러 번 있으면 첫 항목은 유지하고 이후 중복 항목에는 새 ID를 부여한다. 기존 데이터를 조용히 덮어쓰지 않는다.

## 2026-06-19: GitHub Pages 배포는 사용하지 않음

개인 도메인과 Raspberry Pi 운영 환경으로 배포하기 위해 GitHub Pages 배포 workflow를 제거한다. 정적 Vite 앱은 Nginx Docker 이미지로 빌드한다.

## 2026-06-19: Raspberry Pi arm64 Docker 이미지 배포

기존 `jhwan.dev`, `babyweather`와 맞춰 GitHub Actions에서 `linux/amd64`, `linux/arm64` 이미지를 빌드하고 Docker Hub에 push한다. Raspberry Pi에서는 `docker compose`와 업데이트 스크립트로 최신 이미지를 반영한다.
